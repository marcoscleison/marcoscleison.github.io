Interesting Programming Languages: Squirrel

Posted by Marcos Cleison on Wed, Feb 15, 2017 @

Squirrel is an interpreted object-oriented language in the Java, C++ sense.

The language is inspired by several programming language stars. From Lua language, the Squirrel brings the small footprint interpreter, allowing to be used in limited resource hardware (low memory, low bandwidth etc.) making it suitable for real-time applications such as games app.

Squirrel also is inspired by others languages like JavaScript and Python.

The language is suitable to be used in open source and commercial projects because it is licensed under MIT license.

Its scripting nature brings dynamic typing similar to JavaScript with additional features such as delegations higher order functions, lexical scoping, generators, efficient tail recursion etc. The language also possesses classes and inheritance with a Java-like syntax. This reduces the learning curve to newcomers.

Squirrel has a hybrid memory management system structured by the combination of garbage and reference counting approach.

The compiler and the virtual machine is the tiny sized source. They have less than 7K LOC in C++. This results in an executable with size 100kb-150kb.

Tables and Arrays

The first part of the code is a declaration of a local table data type. The table is a data container with the associative key-value structure used to aggregate different data types. This concept is similar to the Lua tables. The array is similar to tables, but they are enclosed by [] and the elements are indexed by an integer.

local table = {
    a = &quot;10&quot;
    subtable = {
               array = [1,2,3]
    },
    [10 + 123] = &quot;expression index&quot;
}
 
local array=[ 1, 2, 3, { a = 10, b = &quot;string&quot; } ];

The Squirrel language allows the iteration over array/tables elements using the 'foreach' keyword.

foreach (i,val in array)
{
    ::print(&quot;the type of val is&quot;+typeof val);
}

I the example above, the for statement iterate over the elements of array getting its values and indexes.

Classes and Object Oriented Programming

The Squirrel language is an Object Oriented Programming language with classic concept of objects, classes, and methods. The classes are declared using the "class" keyword. The constructor of the class is named "constructor" and can possess zero or more parameters. The class fields are created by initialization with a value or using "null" keyword. Differently, from C++ or Php, class fields in Squirrel programming languages are not accessed using a "this" keyword but using directly its name.

class Entity
{    
    constructor(etype,entityname)
    {
        name = entityname;
        type = etype;
    }
                                    
    x = 0;
    y = 0;
    z = 0;
    name = null;
    type = null;
}

Methods

The methods in Squirrel programming language is declared using the "function" keyword, similar to Lua or JavaScript language. There are two forms of the method declaration:

  • Method declaration outside the class definition.
  • Method declaration inside the class definition.

The method declaration outside the class is similar to C++ method implementation in .cpp file:

function Entity::MoveTo(newx,newy,newz)
{
    x = newx;
    y = newy;
    z = newz;
}

The method can be equally declared inside the class as shows the code fragment:

    function DoDomething()
    {
        ::print(&quot;something&quot;);
    }

Class Inheritance

The Squirrel programming language also supports class inheritance. This is accomplished using the 'extends' keyword in a similar way to Java class extending.

 class Player extends Entity {
    constructor(entityname)
    {
        base.constructor(&quot;Player&quot;,entityname)
    }
    function DoDomething()
    {
        ::print(&quot;something&quot;);
    }
    
}

The class that inherits from other class can call parent's constructor using the keyword 'base'. This keyword allows not only the access to the parent's constructor but also to the parent's method.

Imperative Functions

Calling functions is distinguished from methods with the scope"::" operator. So the ::print() denotes that the function print is not a method by a conventional imperative function.
