Interesting languages- IO

Posted by Marcos Cleison on Sun, Jan 29, 2017 @

Iolang

Io is a dynamic and expressive programming language. It is heavily based on Smalltalk, Self, NewtonScript, Act1,Lisp and Lua.

From Smalltalk, Io language brings the idea that "all values are objects". The Self language contributes to Io with the prototype-based inheritance. Act1 was used as inspiration to bring actors and futures to the concurrency.
Io also has modifiable AST and runtime inspection based on Lisp. The interpreter API can be easily used because its design has a similar simplicity of Lua embedding API.

Io language remarkably does not have statement. It possess only expressions that are evaluated along code execution. This feature make Io a language with small syntax as we can see in its core BNF grammar:

exp        ::= { message | terminator }
message    ::= symbol [arguments]
arguments  ::= &quot;(&quot; [exp [ { &quot;,&quot; exp } ]] &quot;)&quot;
symbol     ::= identifier | number | string
terminator ::= &quot;\n&quot; | &quot;;&quot;

The expressions are delimited by new line carriage return or ";". The expression is composed main by an identifier representing an object, string or numeric value followed by a message and optional arguments in the form:

object message (args)

The message is a key concept in Smalltalk based language. It is related to the method idea existent in C++, and Java. The execution of "method" code is started when the "object" receives the message. The messages can have none or more parameters that are sent to the object through message slot. So, in a hello world program in Io should be:

&quot;Hello, world!\n&quot; print

In this example, the "Hello, world!\n" string object (remember: every value is an object, even numbers) receives the message 'print' that exhibits the content of the string.

The Io language has basic assignment arithmetic, conditional and logical operators with infix. Ex.:

a := 10

c := a +3

c print

The code above is parsed as:

  1. The "a" object is created and it receives an assignment message with 10 as parameter. The assignment message sets the value of an object.

  2. The "c" object is created and receives assignment message containing as parameter the value evaluated from "a + 3". The expression "a + 3" is executed as an object receives the sum operation message with 3 as a parameter. The execution of this operation is sent to c.

  3. "c" object, now with value 13, receives a message from print slot informing to display its value.

It is kinda mantra, that "all values are the object" that can receive messages. Even control-flow are not a statement but functions that have condition and code block as parameters. The if the expression can be:

(a == 1) ifTrue (
  writeln(&quot;a is one&quot;)
) ifFalse(
  writeln(&quot;a is not one&quot;)
)

(a = = 1) is a function that returns a Boolean-valued object. This kind of object has the "ifTrue" message and the "ifFalse" message. If the value returned by (a ==1 ) is true, the "ifTrue" message is executed evaluating its parameters expressions (code block). if the value returned is false, the "ifTrue" message is ignored, but the "else" message is sent, evaluating the expression.

The "for" function is used to repeat expressions evaluations. Ex.:

for(i, 1, 10, write(i, &quot; &quot;))

The for function receives four parameters: the counter, the initial value of the range, the final value of range and the expressions to be repeatedly evaluated according to the range values. The code above will print the i value from 1 to 10.

The repetition control while having a similar syntax. The function while receives two parameters: the loop condition and the expression to be executed. Ex.:

a := 1
while(a &lt; 10, 
    a print
    a = a + 1
)

In this example, the function while evaluates two expressions while a value is less than 10.

This syntax is elegant to write object oriented code. An object of type list can be created with the syntax:

//creates an empty list
a := List clone
// append  an item
a append(&quot;b&quot;)

//gets the list size and display it
a size print

The "Classes" are defined from the Object prototype and customized adding field and methods.

//Creates Dog &quot;class&quot; like object
Dog := Object clone
//Creates a field in Dog &quot;class&quot;
Dog name := &quot;Marley&quot;
//Creates the method bark
Dog bark := method(&quot;woof!&quot; print)
//Instanciate a new object from the &quot;class&quot; Dog
bob := Dog clone
//Executes the method bark
bob bark

The function "method" is used to create the method in Io objects. The parameter is a list of expressions that is evaluated when the method is called.

There are others interesting features in this language. To know more about go to http://iolanguage.org/ .
