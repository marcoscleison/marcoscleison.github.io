Introductory Tutorial On How To Use Object Orientation In C

Posted by Marcos Cleison on Sun, Nov 12, 2017 @

Object-oriented programming is often used paradigm and estimated by developers of various aspects of computer science. On the other hand, imperative languages are also heavily used, in particular at medium and low-level system development languages. In this category, the C language stands out as the more popular imperative programming language.

This popularity is due to its frequent use in large and interesting projects. Among these projects, we can mention the Linux, FreeBSD, NetBSD, Minix, Darwin and OpenBSD kernels, the GCC compiler, graphical libraries such as GTK etc.

Some developers would like to have the logical and code organization of an object oriented programming language but in an imperative language such as C.

In this introductory post, we will study how to use object-orientation in C language and how to elaborate techniques in this language that allows some of the object oriented like features.

It should be emphasized that being the C language an imperative language, this has limitations in implementing object orientation, but does not prevent that we get something similar.

“Class” Structure

Our goal is to create a “class” containing two integer fields. For that we define a structure having the public fields.

The next step is to define a type that make reference type to the “struct object” through pointers.

// Type definition
typedef struct object* Object;

Builders

In object-oriented languages, all objects are created by a special procedure called constructor. The constructor has responsibility release and reserve space in memory for the object and initializes its instance. On constructing with success w object is returned as a pointer to the memory where its is stored and can be accessed by objects variables of type Object.

//Constructor
Object newObject() {
	//Allocate memory 
	Object obj = (Object)malloc(sizeof(struct object));
	//Init the field values.
	obj->a = 1;
	obj->b = 2;

	return obj; //returns the built object

}

Methods
