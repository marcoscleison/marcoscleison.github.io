Interesting languages: TCL

Posted by Marcos Cleison on Sun, Jun 19, 2016 @

TCL is an interpreted programming language with minimalist language grammar, dynamic typing, automatic storage allocation and garbage collection and support for native Unicode.

###Introduction

The TCL name means Tool Command Language. As the name states, the aims of the language is serve as an easy scripting programming language to be embedded in others applications or as a toolkit for task automation and fast prototyping.

The language was created by by John Ousterhout in 1988 while working at the University of California, Berkeley. The language acquired a massive user base, mainly in applications that needed an embedded scripting language for configuration and extension. This made the creator wins the ACM Software System Award in 1997.

The TCL programming language have a small syntax grammar. The language is composed mainly by a command and a variadic arguments that is formed by strings, numbers or other commands.

command arg1 arg2 arg3 ... argn

Hello World

Let's see a simple example:

print &quot;Hello World&quot;

This simple example is formed by a command print and an string argument &quot;Hello World&quot;. The print command is a built-in command that prints its arguments.

####Variable Assignment and Substitution

Usually, variable assignment is obtained using the classic syntax variable = value. However, in TCL programming language the variable assignment is obtained using the command set with syntax:

set variable values

If we want a variable x with value 10 we would code in tcl as:

set x 10

Let's print the x variable:

print x

The output is:

x

What is up? Why I set x with value 10 but cannot print it? this is not a but,it is a tcl feature. The arguments of the commands are considered string, even though they does not have "". So, so this command print x is interpreted as print the string "x". To print or access the value of variable x, it is necessary to use the substitution value. This is accomplished prefixing the variable name with $.

print $x

The out put will be:

10

The print command will replace the argument $x by the value of variable x. This process is called substitution.

Multiword string without substitution can be expressed using {}.

print {The $x will not be substituted}

On the other hand, sometimes we need to pass as argument the result of some command. To do that, we can use the [], with the sintax

command1 [command2 args1 args2 args3]

The command1 will receives the result of the command2 evaluation.
