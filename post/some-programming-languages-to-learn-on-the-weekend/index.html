Terra

C languages uses macro preprocessor to allow some control over the code logic. C++ uses the powerful template definitions to allow metaprogramming. However, C++ templates can be complex and inflexible to some voluble metaprogramming form. This issue is is resolved in the Terra language.

Terra means "earth" in Portuguese language. This name comes from its counterpart and auxiliary language Lua (which means moon in Portuguese).

Terra is a fast low level language that uses the lua language as template and macro generator for impressive metaprogramming. This languages uses a lua like syntax and is perfectly compatible with luavm .

Similarly to C++, terra does not have garbage collector, its memory is manually managed. However, terra uses lua as template language from the beginning.

The language have core operators, function calls, types and control-flow surrounded by lua language. This feature makes terra a language with powerful metaprogramming. Lets see an example:


terra addone(a : int)
    return a + 1
end

print(addone(2)) --this outputs: 3

This example shows a terra function called "addone" that receives en integer parameter and add one to its value returning it. The value is sent to the output.

To differentiate terra function from lua functions, the language prefixes its function declaration with "terra" keyword.

The language allows to call native function using the lua function "terralib.includec".

local C = terralib.includec(&quot;stdio.h&quot;)
terra main()
    C.printf(&quot;hello, world\n&quot;)
end

main()

In this example, the stdio.h C functions are imported and scoped to the loacl lua contruct (local C). The terra function uses the well known function printf to display the message.

This code can be executed using the Terra jit. The terra jit is built upon the LLVM compiler construction framework. However, jit is not the only model of terra execution. The language also allows to compile the code to object file .o and also as an executable. The compilation parameters is defined in the source code using lua language.

- save a .o file you can link to normal C code:
terralib.saveobj(&quot;hello.o&quot;,{ main = main })

-- save a native executable
terralib.saveobj(&quot;hello&quot;, { main = main })

The terra language can be easily embedded in third party applications. Its provides an API similar to Lua language:

#include &lt;stdio.h&gt;
#include &quot;terra.h&quot;

int main(int argc, char ** argv) {
    lua_State * L = luaL_newstate(); //create a plain lua state
    luaL_openlibs(L);                //initialize its libraries
    //initialize the terra state in lua
    terra_init(L);
    for(int i = 1; i &lt; argc; i++)
        //run the terra code in each file
        if(terra_dofile(L,argv[i]))  
            exit(1);
    return 0;
}
```,

# Io

Io is a dynamic and expressive prograaming language. It is heavily based on Smalltalk, Self, NewtonScript, Act1,Lisp and Lua.

From Smalltalk, Io language brings the &quot;all values are objects&quot;. The Self language contributes to Io with the prototype-based inheritance. Act1 was used as inspiration to bring actors and futures for concurrency.   
Io also have modifiable AST and runtime inspection based on Lisp. The interpreter API can be easily used because its design have a similar simplicity of Lua embedding  API. 

Io language remarkably does not have statement. It possess only expressions that are evaluated along code execution. This feature make Io a language with small syntax  as we can see in its BNF grammar:
```bnf
exp        ::= { message | terminator }
message    ::= symbol [arguments]
arguments  ::= &quot;(&quot; [exp [ { &quot;,&quot; exp } ]] &quot;)&quot;
symbol     ::= identifier | number | string
terminator ::= &quot;\n&quot; | &quot;;&quot;

The expressions is delimited by new line carriage return or ";". The expression is composed main by an identifier representing an object, string or numeric value followed by a message and optional arguments in the form:

object message (args)

The message is a key concept in smalltalk based language. It is related to the method idea existent in C++, and Java. The execution of "method" code is started when the "object" receives the message. The messages can have none or more parameters that is sent to the object through message slot. So, in a hello world program in Io should be:

&quot;Hello, world!\n&quot; print

In this example, the "Hello, world!\n" string object (remeber:every value is an object, even numbers) receives the message 'print' that exhibits content of string.

The Io language have basic assignment arithmetic, conditional and logical operators with infix. Ex.:

a := 10

c := a +3

c print

The code above is parsed as:

  1. a object is created and its receives an assignment message with 10 as parameter. The assignment message sets the value of object.

  2. c object is created and receives assignment message containing as parameter the value evaluated from "a + 3". The expression "a + 3" is executed as a object receives the sum operation message with 3 as parameter. The execution of this operation is sent to c.

  3. c object, now with value 13, receives a message from print slot informing to display its value.

It is kind a mantra, that all values are object that can receive messages. Even control-flow are not statement but functions that have condition and code block as parameters. The if expression can be:

(a == 1) ifTrue (
  writeln(&quot;a is one&quot;)
) ifFalse(
  writeln(&quot;a is not one&quot;)
)

(a = = 1) is a function that returns a Boolean valued object. This kind of object have the "ifTrue" message and the "ifFalse" message. If the value returned by (a ==1 ) is true, the "ifTrue" message is executed evaluating its parameters expressions (code block). if the value returned is false, the "ifTrue" message is ignored, but the "else" message is sent, evaluating the expression.

The "for" function is used to repeat expressions evaluations. Ex.:

for(i, 1, 10, write(i, &quot; &quot;))

The for function receives four parameters: the counter, the initial value of range, the final value of range and the expressions to be repeatedly evaluated according to the range values. the code above will print the i value from 1 to 10.

The repetition control while have a similar syntax. The function while receives two parameters: the loop condition and the expression to be executed. Ex.:

a := 1
while(a &lt; 10, 
    a print
    a = a + 1
)

In this example the function while evaluates two expressions while a value is less than 10.

This syntax is elegant to write object oriented code. An object of type list can be created with syntax:

//creates an empty list
a := List clone
// append  an item
a append(&quot;b&quot;)

//gets the list size and display it
a size print

The "Classes" are defined from the Object prototype and customized adding field and methods.

//Creates Dog &quot;class&quot; like object
Dog := Object clone
//Creates a field in Dog &quot;class&quot;
Dog name := &quot;Marley&quot;
//Creates the method bark
Dog bark := method(&quot;woof!&quot; print)
//Instanciate a new object from the &quot;class&quot; Dog
bob := Dog clone
//Executes the method bark
bob bark

The function "method" is used to create method in Io objects. The parameter is a list of expressions that is evaluated when method is called.

There are others interesting features in this language. To know more about go to http://iolanguage.org/ .

Squirrel

Squirrel is an interpreted object oriented language in the Java, C++ sense.

The language is inspired by several programming language stars. From Lua language, the Squirrel brigs the small footprint interpreter, allowing to be used in limited resource hardware (low memory, low bandwidth etc.) making it suitable to real-time applications such as games app.

Squirrel also is inspired by others languages like JavaScript and Python.

The language is suitable to be used in open source and commercial projects because it is licensed as MIT licence.

Its scripting nature brings dynamic typing similar to javascript with additional features such as delegations higher order functions, lexical scoping,generators, efficient tail recursion etc. The language also possesses classes and inheritance with a Java like syntax. This reduces the learning curve to new comers.

Squirrel has an hybrid memory management system structured by the combination of garbage and reference counting approach.

The compiler and the virtual machine is tiny sized source. They have less than 7 LOC in C++. This results in an executable with size 100kb-150kb.

Necko

Necko is a programming language that runs on NeckVM. Its primary aims is to serve as a target language platform for new high level language. In other words, necko language can be used as compiler beckend.

The language possesses a compiler that translate from necko source code to bytecode and a small virtual machine to execute it. After installation, you can execute necko bytecodes with command:

necko app

Each compiled necko file has the extension .n and contains the language modules. The VM can use necko based dynamic libraries. These libraries have extension .ndll by its internal structures are.
